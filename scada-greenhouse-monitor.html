SCADA Greenhouse Monitor

Python FastAPI MySQL Avalonia

Overview

GardenTracker is a comprehensive greenhouse monitoring and control system that enables real-time environmental sensor monitoring with customizable alert thresholds for temperature, humidity, and actuators like water pumps. The system features local automation rules and a fully customizable display interface tailored to user needs.

Key Features

  • Real-time sensor data visualization with dynamic card rendering and status indicators
  • Local automation with database-synced thresholds and automated scheduling
  • Historical data graphing and CSV export capabilities
  • Full hardware integration with Windows stubbing fallback for development
  • Fully customizable card-based interface with drag-and-drop editor
  • Persistent settings between sessions with error recovery
  • Strong accessibility and user experience design
  • Client-level automation rules that submit requests on behalf of users

Technical Implementation

Frontend built with .NET 9 and Avalonia UI framework for cross-platform desktop support, featuring ScottPlot for data visualization. Backend powered by FastAPI with Python for high-performance async operations and MySQL for time-series data persistence.

The system implements intelligent hardware detection with automatic stubbing - when physical hardware is connected, it runs on actual sensors; otherwise, it fully recreates the environment for development and testing. Hardware setup includes Raspberry Pi GPIO integration with ADS1115 ADC for analog sensor readings.

Architecture supports RESTful API communication, automated threshold monitoring at the database layer, and persistent configuration management across sessions.
